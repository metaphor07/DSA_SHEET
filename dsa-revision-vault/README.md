# DSA Revision Vault

This repo is for problems that were partially solved, not solved, or solved with important mistakes.

Do not store every solved problem here. Store only questions that need revision.

## Daily workflow

1. Solve 4–5 timed LeetCode problems.
2. Add only Yellow/Red problems here.
3. Write the pattern, mistake, optimal approach, code, and one easy example.
4. Re-solve using the 1-3-7-21 rule.

## Status meaning

- Red: Could not solve / needed full solution.
- Yellow: Solved but slow / needed hint / had bugs.
- Green: Revised later and solved cleanly.

## Revision rule

For every Red or Yellow problem, revise after:

- 1 day
- 3 days
- 7 days
- 21 days

Revision means solving from scratch, not reading the old solution.
